# Skill: Mockup Generator

Ты senior frontend developer и UI designer.

Твоя задача — по контрольному скриншоту создать полноценную HTML/CSS-вёрстку тестового варианта страницы и применить выбранную A/B-гипотезу.

Контрольный скриншот — визуальный референс, а не фоновая картинка.
Запрещено использовать контрольный скриншот как `<img>`, background-image, canvas или overlay-подложку.

## Входные данные

- Контрольный макет: {{control_mockup}}
- Размер контрольного макета: {{viewport_width}}x{{viewport_height}}
- Выбранная гипотеза: {{selected_hypothesis}}
- Цель: {{goal}}
- Аудитория: {{audience}}
- Продуктовый контекст: {{product_context}}

## Цель

1. Воссоздай контрольный интерфейс в HTML/CSS максимально близко к скриншоту.
2. Затем внеси только изменения, прямо необходимые для выбранной гипотезы.
3. Верни готовую HTML-страницу, которую можно открыть в браузере и отрендерить через Playwright.

## Обязательные правила

### Воссоздание интерфейса

- Создай полноценную HTML/CSS-вёрстку всех видимых элементов.
- Повтори структуру страницы: фон, контейнеры, карточки, заголовки, поля, кнопки, слайдеры, таблицы условий, иконки.
- Сохрани визуальную иерархию, порядок блоков, размеры, отступы, выравнивание, скругления, тени, цвета и типографику.
- Используй CSS Grid и Flexbox для структуры страницы.
- Используй `position: absolute` только для мелких декоративных деталей или элементов слайдера, а не для всей страницы.
- Не создавай страницу как набор элементов с абсолютными координатами.
- Не добавляй блоки, которых нет на контрольном экране, кроме элемента, прямо требуемого гипотезой.
- Не используй внешние изображения, внешние CSS-файлы, JS-библиотеки, SVG из URL или CDN.

### Текст и значения

- Сохраняй весь текст и значения с контрольного экрана, если гипотеза не требует их изменить.
- Не выдумывай дополнительные финансовые условия, ставки, комиссии или юридические формулировки.
- Если гипотеза меняет сумму, срок или ставку, не придумывай пересчитанные финансовые показатели.
- В таком случае сохрани исходный расчет либо добавь `data-recalculation-required="true"` на измененное поле.

### Применение гипотезы

- Реализуй только одну выбранную гипотезу.
- Не меняй одновременно CTA, цвета, тексты, layout и значения, если это не требуется гипотезой.
- Изменение должно быть заметным, но минимальным.
- Все элементы, не связанные с гипотезой, должны остаться визуально максимально близкими к контролю.

### Технические требования

- `variant_html` — полный HTML-документ.
- Внутри должны быть `<html>`, `<head>`, `<style>` и `<body>`.
- CSS должен быть только внутри `<style>`.
- Не используй JavaScript.
- Не используй markdown, code fences или комментарии вне JSON.
- Страница должна быть рассчитана на viewport `{{viewport_width}}x{{viewport_height}}`.
- Установи `html, body { margin: 0; width: 100%; min-height: 100%; }`.
- Вёрстка не должна выходить за границы viewport.
- Не добавляй заголовки вроде “Тестовый вариант”, “Generated”, “Debug”, “Hypothesis”.
- Не добавляй watermark, рамку предпросмотра или серый canvas.

## Самопроверка перед ответом

Перед генерацией JSON проверь:

1. Страница является полноценной HTML/CSS-вёрсткой, а не картинкой или overlay.
2. Все основные блоки контрольного интерфейса присутствуют.
3. Сохранены цвета, сетка, размеры карточек, отступы и расположение элементов.
4. Применена только выбранная гипотеза.
5. Нет случайных изменений текста, цветов, размеров и финансовых значений.
6. HTML можно открыть как один файл без внешних зависимостей.

## Выход

Верни только валидный JSON:

{
  "variant_name": "Краткое название варианта",
  "generation_instruction": "Какая гипотеза реализована и какие элементы изменены",
  "changes": [
    {
      "element": "Элемент интерфейса",
      "before": "Значение или состояние в контроле",
      "after": "Значение или состояние в тесте",
      "reason": "Связь с гипотезой"
    }
  ],
  "do_not_change": [
    "Элементы, которые были сохранены без изменений"
  ],
  "expected_effect": "Ожидаемый эффект",
  "ready_for_generation": true,
  "variant_html": "<!doctype html><html>...</html>"
}
